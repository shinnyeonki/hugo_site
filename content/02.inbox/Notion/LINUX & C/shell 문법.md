```
위치 매개 변수
$0 명령
$1 첫번째 명령
...

$@, $* 모든 인자
$@는 환경변수 $IFS 에 설정된 구분자를 통해 위치 매개변수를 분리하여 보관
$*는 모든 매개변수를 하나의 스트링으로 보관ㄷㅊ 
$# 인자의 개수 (ls 명령의 경우 인자가 0)
$$ 셸 스크립트의 PID
$? 실행결과 값(파일 디스크립터의 숫자 ex)stdin stdout stderr 등등)
$- 옵션 플래그 출력
```

```
{ 와 ( 의 차이
쉘 스크립트에서 (parentheses)와 {braces}는 다른 목적과 사용법을 가지고 있습니다. 각각의 차이점을 설명해드리겠습니다.

(Parentheses): 비로그인 셸

(와 ) 사이에 있는 명령어나 표현식은 서브셸로 실행됩니다. 서브셸은 현재 쉘 스크립트와 별개의 새로운 프로세스로 실행되며, 이로 인해 변수의 유효 범위 등이 영향을 받을 수 있습니다.
서브셸을 사용하면 원본 쉘 스크립트와는 독립된 환경에서 명령어를 실행할 수 있습니다. 예를 들어, (command1; command2)와 같이 사용하면 command1과 command2가 서브셸에서 순차적으로 실행됩니다.
{Braces}:

{와 } 사이에 있는 명령어나 표현식은 현재 쉘 스크립트에서 실행됩니다. 즉, 새로운 서브셸을 만들지 않습니다.
주로 제어 구조(조건문, 반복문 등) 내에서 사용되어 코드 블록을 그룹화하거나, 변수 스코프를 제어하는 데 사용될 수 있습니다. 예를 들어, { command1; command2; }와 같이 사용하여 여러 명령어를 하나의 블록으로 묶을 수 있습니다.
```
